Configuration

Essentiel

Configurer les informations utilisateur

Essentiel pour identifier l'auteur de chaque commit dans l'historique du projet.

# Définit votre nom pour tous les commits
git config --global user.name "Votre Nom"

# Configure l'email associé à vos contributions
git config --global user.email "votre@email.com"

Configurer l'éditeur par défaut

Personnalise l'expérience de rédaction des messages de commit avec votre éditeur préféré.

# Utilise VS Code comme éditeur par défaut
git config --global core.editor "code --wait"

Lister toutes les configurations

Permet de vérifier rapidement toutes les configurations Git actives sur votre système.

# Affiche l'ensemble des paramètres configurés
git config --list

Configurer des alias

Accélère votre workflow en créant des raccourcis pour les commandes fréquemment utilisées.

# Crée des alias pour les commandes courantes
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

Dépôts

Fondamental

Initialiser un nouveau dépôt

Première étape pour commencer à suivre un projet avec Git.

# Transforme le répertoire courant en dépôt Git
git init

Cloner un dépôt existant

Permet de récupérer une copie complète d'un projet existant, avec tout son historique.

# Clone un dépôt dans un dossier du même nom
git clone https://github.com/user/repo.git

# Clone dans un dossier spécifique
git clone https://github.com/user/repo.git dossier-cible

Afficher les dépôts distants

Visualise les connexions entre votre dépôt local et les dépôts distants.

# Liste les dépôts distants avec leurs URLs
git remote -v

Ajouter un dépôt distant

Établit une connexion avec un serveur distant pour partager votre code.

# Associe un dépôt distant nommé "origin"
git remote add origin https://github.com/user/repo.git

Modifications

Quotidien

Vérifier l'état des fichiers

Commande fondamentale pour voir quels fichiers ont été modifiés et quel est leur statut.

# Affiche l'état actuel du dépôt
git status

Ajouter des fichiers à l'index

Prépare les modifications pour le prochain commit en les plaçant dans la zone de staging.

# Ajoute un fichier spécifique
git add fichier.txt

# Ajoute tous les fichiers modifiés ou supprimés
git add .

# Ajoute tous les fichiers, y compris les nouveaux
git add -A

Commiter les modifications

Enregistre de façon permanente les modifications dans l'historique du projet.

# Crée un commit avec un message descriptif
git commit -m "Message de commit"

# Ajoute automatiquement les fichiers modifiés
git commit -a -m "Message"

Modifier le dernier commit

Permet de corriger une erreur dans le dernier commit sans créer un nouveau commit.

# Change le message du dernier commit
git commit --amend -m "Nouveau message"

# Ajoute des modifications sans changer le message
git commit --amend --no-edit

Branches

Avancé

Lister les branches

Donne une vue d'ensemble des branches disponibles dans le projet.

# Liste les branches locales
git branch

# Liste toutes les branches (locales et distantes)
git branch -a

# Liste les branches avec le dernier commit
git branch -v

Créer une branche

Permet de développer de nouvelles fonctionnalités sans affecter la branche principale.

# Crée une nouvelle branche sans y basculer
git branch nouvelle-branche

# Crée une nouvelle branche et y bascule
git checkout -b nouvelle-branche

Changer de branche

Bascule entre différentes versions ou fonctionnalités du projet.

# Bascule vers une branche existante (ancienne méthode)
git checkout branche-existante

# Bascule vers une branche existante (nouvelle méthode)
git switch branche-existante

Fusionner des branches

Intègre les modifications d'une branche dans une autre.

# Fusionne une branche dans la branche actuelle
git merge autre-branche

# Fusionne en créant un commit de fusion
git merge --no-ff autre-branche

Historique

Debug

Afficher l'historique

Explore l'évolution du projet à travers ses commits.

# Affiche l'historique complet
git log

# Affiche les 5 derniers commits
git log -n 5

# Format condensé (une ligne par commit)
git log --oneline

# Historique sous forme de graphe
git log --graph

Voir un commit spécifique

Examine en détail les modifications apportées par un commit particulier.

# Affiche les détails d'un commit
git show commit-hash

Bonnes pratiques Git

Pro

Règles d'or des commits

1. Messages explicatifs :

# ❌ À éviter
git commit -m "Corrections"

# ✅ Bonne pratique
git commit -m "Fix: résoudre le bug de calcul dans checkout.js"

2. Commits atomiques :

# Un commit = une modification logique
git add src/utils/calculator.js
git commit -m "Feat: ajouter la fonction de calcul TVA"

3. Convention de messages :

<type>: <description>

[body optionnel]

Types recommandés :
- feat : Nouvelle fonctionnalité
- fix : Correction de bug
- docs : Documentation
- style : Formatage
- refactor : Réorganisation du code
- test : Tests

Workflow recommandé

1. Avant de commencer :

# Mettre à jour en local
git pull --rebase origin main

2. Pendant le développement :

# Ajouter les changements par portions
git add -p

# Vérifier son commit avant validation
git commit -v

3. Avant de pusher :

# Relire l'historique
git log --oneline origin/main..HEAD

# Vérifier les modifications
git diff origin/main
